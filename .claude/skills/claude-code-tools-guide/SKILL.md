# دليل الأدوات الموثوقة لتعزيز Claude Code

## مقدمة: لماذا هذه السبع فقط؟

هذا الدليل يضم **سبع أدوات فقط** جرى التحقق من كل واحدة منها يدويًا: وجود المستودع، صاحبه الحقيقي، نشاط التطوير، وطريقة التثبيت. أثناء البحث ظهرت أدوات أخرى متداولة بكثرة — أبرزها **Awesome Claude Code** و **GSD** — لكنها **استُبعدت عمدًا** من قائمة التوصيات لأننا لم نتمكن من التحقق منها بنفس المستوى: إما أن الاسم يشير لأكثر من مستودع متنافس دون جهة أصلية واضحة، أو أن الرابط المتداول لا يطابق مستودعًا مستقرًا يمكن ضمانه. القاعدة هنا: **لا يُدرج في هذا الدليل إلا ما تحققنا من رابطه الدقيق ومصدره**.

---

## 1. Claude Mem — ذاكرة دائمة عبر الجلسات

- **الرابط الدقيق:** https://github.com/thedotmack/claude-mem
- **المشكلة التي تحلها:** Claude Code ينسى كل شيء بمجرد إغلاق الجلسة أو ضغط السياق (compaction). كل جلسة جديدة تبدأ من الصفر، وتضطر لإعادة شرح المشروع وقراراتك السابقة.
- **آلية العمل:** يلتقط ملاحظات من استخدام الأدوات أثناء الجلسة، يضغطها بالذكاء الاصطناعي إلى ملخصات مركزة، ويخزنها محليًا، ثم يسترجع الملائم منها تلقائيًا في الجلسات القادمة عبر hooks تعمل عند بدء الجلسة.
- **الفائدة لمستخدم Claude Code:** استمرارية حقيقية — يتذكر Claude قرارات المعمارية، والأخطاء التي أصلحتها، وسياق المشروع دون أن تعيد الشرح في كل مرة.

## 2. Superpowers — منهجية تطوير كاملة للوكيل

- **الرابط الدقيق:** https://github.com/obra/superpowers
- **المشكلة التي تحلها:** Claude Code بدون منهجية يقفز مباشرة للكود، فيتخطى التصميم والاختبارات والمراجعة، وتتفاوت جودة النتائج من جلسة لأخرى.
- **آلية العمل:** مجموعة سكيلات قابلة للتركيب تفرض سير عمل منضبطًا: عصف ذهني وتصميم أولًا، ثم خطة تنفيذ، ثم تطوير موجه بالاختبارات (TDD)، ثم مراجعة كود — وينشّط السكيل المناسب تلقائيًا حسب مرحلة العمل.
- **الفائدة لمستخدم Claude Code:** ترقية نوعية في انضباط الوكيل؛ مخرجات أقرب لعمل فريق هندسي محترف بدل «كود أول محاولة».

## 3. Obsidian Skills — سكيلات Obsidian الرسمية

- **الرابط الدقيق:** https://github.com/kepano/obsidian-skills
- **المشكلة التي تحلها:** تعامل الوكلاء مع قواعد معرفة Obsidian (روابط داخلية، Bases، JSON Canvas) يكون عشوائيًا ويكسر تنسيقات الملفات الخاصة.
- **آلية العمل:** سكيلات رسمية منشورة من Steph Ango (الرئيس التنفيذي لـ Obsidian) تعلّم الوكيل صيغ Obsidian المفتوحة و«Obsidian CLI»، وتُثبَّت داخل مجلد `.claude` في جذر القبو (vault) نفسه.
- **الفائدة لمستخدم Claude Code:** يصبح Claude مساعدًا حقيقيًا داخل قاعدة معرفتك — ينشئ ويعدل ويربط الملاحظات بالصيغ الصحيحة، بدعم رسمي من صانع الأداة.

## 4. LightRAG — استرجاع معرفي بالرسم البياني

- **الرابط الدقيق:** https://github.com/HKUDS/LightRAG
- **المشكلة التي تحلها:** البحث المتجهي (vector RAG) التقليدي يفقد العلاقات بين المفاهيم في المستندات الكبيرة، وبدائل مثل GraphRAG ثقيلة ومكلفة.
- **آلية العمل:** إطار RAG خفيف بمعمارية مزدوجة: رسم معرفي (knowledge graph) يلتقط العلاقات بين الكيانات + فهرسة متجهية للاسترجاع الدلالي، مع خادم API وواجهة ويب جاهزة.
- **الفائدة لمستخدم Claude Code:** تربطه بمشروعك أو وثائقك فيجيب Claude من فهم مترابط للمحتوى (من يعتمد على ماذا، ما علاقة كذا بكذا) بدل مقاطع نصية منفصلة.

## 5. Everything Claude Code — ترسانة إعدادات جاهزة

- **الرابط الدقيق:** https://github.com/affaan-m/everything-claude-code
- **المشكلة التي تحلها:** بناء منظومة agents/skills/hooks/commands ناضجة لـ Claude Code يستهلك شهورًا من التجربة والخطأ.
- **آلية العمل:** مجموعة كاملة من إعدادات فائز بهاكاثون Anthropic صُقلت على مدى أشهر من الاستخدام اليومي: وكلاء فرعيون (مثل مراجع كود)، سكيلات، hooks بـ Node.js لحفظ واسترجاع السياق، وأوامر مخصصة — تنسخها إلى مجلد إعدادات Claude لديك.
- **الفائدة لمستخدم Claude Code:** نقطة انطلاق احترافية فورية + دليل مطول في تحسين استهلاك التوكنز والذاكرة والتوازي.

## 6. UI UX Pro Max — ذكاء تصميم واجهات

- **الرابط الدقيق:** https://github.com/nextlevelbuilder/ui-ux-pro-max-skill
- **المشكلة التي تحلها:** الواجهات التي يولدها الذكاء الاصطناعي تبدو عامة ومتشابهة، بألوان وخطوط غير مدروسة لمجال المنتج.
- **آلية العمل:** سكيل يحلل متطلبات مشروعك ويطابقها مع قاعدة بيانات تضم 84 نمط واجهة، و192 لوحة ألوان، و74 توليفة خطوط، عبر أكثر من 160 قاعدة استدلال مرتبطة بمجالات الصناعة، ليخرج بنظام تصميم متكامل.
- **الفائدة لمستخدم Claude Code:** واجهات بهوية بصرية مقصودة ومناسبة للمجال من أول توليد، بدل تعديلات جمالية لا تنتهي.

## 7. n8n-MCP — أتمتة سير العمل عبر n8n

- **الرابط الدقيق:** https://github.com/czlonkowski/n8n-mcp
- **المشكلة التي تحلها:** بناء تدفقات أتمتة في n8n يدويًا يتطلب معرفة مئات العقد (nodes) وخصائصها، وClaude لا يعرف تفاصيلها الدقيقة من تدريبه.
- **آلية العمل:** خادم MCP يمنح Claude وصولًا موثقًا لأكثر من 2,100 عقدة n8n (أساسية ومجتمعية) بخصائصها وعملياتها، فيستطيع فهم وبناء وتعديل تدفقات n8n برمجيًا.
- **الفائدة لمستخدم Claude Code:** تصف الأتمتة التي تريدها بلغة طبيعية، ويبني Claude تدفق n8n صحيحًا وجاهز التشغيل — ربط خدمات، جدولة، إشعارات، وغيرها.

## كيفية التثبيت

| الأداة | أمر التثبيت |
|---|---|
| Claude Mem | `npx claude-mem install` |
| Superpowers | `/plugin install superpowers@claude-plugins-official` (من داخل Claude Code) |
| Obsidian Skills | `npx skills add https://github.com/kepano/obsidian-skills` (داخل جذر القبو) |
| LightRAG | `pip install "lightrag-hku[api]"` أو Docker Compose |
| Everything Claude Code | استنساخ المستودع ونسخ agents/skills/commands إلى مجلد إعدادات Claude، ودمج hooks في `settings.json` |
| UI UX Pro Max | `npm install -g ui-ux-pro-max-cli && uipro init --ai claude` أو `/plugin marketplace add nextlevelbuilder/ui-ux-pro-max-skill` |
| n8n-MCP | خدمة سحابية عبر dashboard.n8n-mcp.com أو تشغيل ذاتي بـ NPX / Docker |

## ملخص التحقق من الحقائق

جرى التحقق من الروابط السبعة مباشرة من GitHub (يوليو 2026): كلها مستودعات قائمة ونشطة، وأصحابها هم الجهات الأصلية للأدوات — بما فيها التأكد أن `kepano/obsidian-skills` هو المستودع الرسمي من الرئيس التنفيذي لـ Obsidian، وأن `affaan-m/everything-claude-code` هو المستودع الأصلي لصاحب المشروع (لا إحدى نسخه المتفرعة الكثيرة). أما Awesome Claude Code وGSD فلم يُدرجا لتعذر ضمان مستودع أصلي وحيد موثوق لهما.
